# 性能对比示例

对比 JavaScript 和 WebAssembly 在计算密集型任务中的性能差异。

## 文件说明

- `fibonacci.wat` - WebAssembly 文本格式源代码
- `fibonacci.wasm` - 编译后的二进制文件（需要编译）
- `index.html` - 性能对比演示页面

## 编译步骤

```bash
wat2wasm fibonacci.wat -o fibonacci.wasm
```

## 运行示例

1. 启动本地 HTTP 服务器
2. 在浏览器中打开 `http://localhost:8000/03-performance/`

## 测试内容

### 1. 斐波那契数列（递归）
- 展示递归算法的性能差异
- ⚠️ 注意：递归版本计算较大数字会很慢

### 2. 斐波那契数列（迭代）
- 展示迭代算法的性能差异
- 可以测试更大的数字

### 3. 阶乘计算
- 展示循环计算的性能差异

### 4. 数组求和
- 展示批量数据处理的性能差异
- 演示内存操作性能

## 性能说明

WebAssembly 的性能优势主要体现在：

1. **静态类型**：编译时优化
2. **二进制格式**：加载和解析更快
3. **接近机器码**：执行效率高

**注意**：性能提升取决于：
- 任务类型（计算密集型 vs I/O密集型）
- 浏览器实现
- 硬件环境
- 代码优化程度

通常 WebAssembly 在计算密集型任务中可以达到 JavaScript 的 1.5-3 倍性能。
